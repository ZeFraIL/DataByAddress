## 🎯 Цель проекта
• **Какую задачу решает приложение:** Позволяет получать географические координаты (широту и долготу) и подробную информацию об адресе по введенным текстовым данным (улица, город, страна). Также предоставляет возможность просмотреть адрес на карте.  
• **Почему эта задача важна:** Геокодирование необходимо для логистических приложений, сервисов доставки и инструментов сбора данных, где требуется точное местоположение.  
• **Целевая аудитория:** Пользователи, которым нужно быстро узнать координаты места или получить подробные адресные данные.  

________________________________________
## 📌 Требования к приложению
### Функциональные требования
• Ввод адреса (улица, город, страна).  
• Переход к внешнему приложению карт для отображения местоположения.  
• Получение полных данных об адресе (индекс, регион, номер дома и т.д.) через Geocoder.  
• Сохранение полученных координат в локальную БД SQLite.  
• Синхронизация данных с Firebase Realtime Database.  

### Нефункциональные требования
• **Производительность:** Быстрый отклик интерфейса при выполнении сетевых запросов.  
• **Удобство использования:** Минималистичный и понятный интерфейс с тремя полями ввода и кнопками действий.  
• **Надёжность:** Обработка ошибок при отсутствии интернета или неверно введенном адресе.

## 🧠 Общая архитектура
• **Выбранный подход:** MVC (Model-View-Controller).  
• **Почему выбран именно он:** Для небольшого проекта этот подход обеспечивает достаточную структурированность без избыточности кода.  
• **Основные компоненты системы:**  
  - **View:** XML-разметки (activity_main.xml).  
  - **Controller:** MainActivity.java (управляет логикой и UI).  
  - **Model:** LatLng.java (представление данных), HelperDB.java (работа с данными).

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:** Главный экран и контроллер приложения.  
**Ответственность:** Обработка пользовательского ввода, взаимодействие с Google Geocoder, управление базой данных и Firebase.  
**Основные методы:**  
- `onCreate()` — инициализация компонентов и установка слушателей кликов.  
- `initComponents()` — привязка элементов UI и инициализация БД.  
- `date_and_time()` — формирование метки времени для записи в Firebase.  
**Взаимодействие с другими классами:** Использует `HelperDB` для записи в SQLite и `LatLng` для передачи данных в Firebase.  

### 📌 Класс: LatLng
**Роль:** Модель данных.  
**Почему используется:** Для удобной упаковки координат (latitude, longitude) при отправке в Firebase.  

### 📌 Класс: HelperDB
**Роль:** Слой данных (SQLite).  
**Ответственность:** Создание таблицы и управление структурой локальной базы данных.

## 🔄 Диаграмма работы приложения
1. Пользователь вводит адрес в поля ввода.  
2. Нажимает кнопку "Data by address".  
3. Приложение вызывает Geocoder.  
4. Результаты выводятся в TextView.  
5. Данные сохраняются в SQLite и отправляются в Firebase.

## ⚙️ Работа с потоками
• **Используются ли:** Geocoder работает синхронно в главном потоке (в текущей реализации), что может приводить к микро-зависаниям. Рекомендуется перенос в фоновый поток (Coroutines/AsyncTask).  
• **Почему выбран этот способ:** Простота реализации для учебного проекта.  
• **Как предотвращаются:**  
  – **Зависания (ANR):** В текущей версии следует быть осторожным с долгими запросами.  
  – **Утечки памяти:** Использование контекста приложения там, где это необходимо.  

________________________________________
## 💾 Работа с данными
• **Где хранятся данные:** Локально в SQLite (`all_data.db`) и удаленно в Firebase.  
• **Почему выбран этот способ:** SQLite обеспечивает оффлайн-доступ, Firebase — облачную синхронизацию.  
• **Как обеспечивается:**  
  – **сохранность:** Использование надежных механизмов SQLiteOpenHelper.  
  – **корректность:** Проверка наличия данных перед записью.  

________________________________________
## 🌐 Работа с сетью
• **Как выполняются запросы:** Через Google Geocoder API и Firebase SDK.  
• **Обработка ошибок:** Блоки try-catch для отлова IOException при запросах к Geocoder.  
• **Что происходит при отсутствии интернета:** Выводится сообщение об ошибке в TextView ("Ошибка: ...").
